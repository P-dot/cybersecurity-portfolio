# Lab 06 — Defensive Python for Linux Log Review and SOC Reporting

## Objective

Use Python to automate basic defensive review of Linux log evidence generated during Lab 05.

## Scope

This lab focuses on defensive automation, log review and reporting.

It does not perform exploitation, credential theft, malware execution, evasion, persistence, brute force or unauthorized activity.

## Initial evidence sources

The first Python scripts will read evidence files generated during Lab 05:

- lab05_02_auth_sudo_sessions.txt
- lab05_03_system_health_errors.txt
- lab05_04_mini_triage_soc.md
- lab05_05_final_closure_summary.md

## Planned missions

- Mission 6.1 — Lab initialization and Python check.
- Mission 6.2 — Read Lab 05 evidence files with Python.
- Mission 6.3 — Extract defensive patterns from logs.
- Mission 6.4 — Generate an automated Markdown report.
- Mission 6.5 — Build a small event classifier.
- Mission 6.6 — Final professional closure.
